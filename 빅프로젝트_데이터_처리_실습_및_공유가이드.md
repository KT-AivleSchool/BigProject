# 💻 빅프로젝트 데이터 처리 로컬 실습 및 결과 공유 가이드

오늘의 목표는 **모든 팀원이 담당한 공공데이터셋을 로컬 환경에서 직접 정제 및 공간 데이터 변환 처리를 해보고, 그 결과와 데이터 품질 상태를 공유하는 것**입니다.

이 가이드는 로컬 개발 환경(Python/Pandas/GeoPandas)에서 데이터를 직접 다루어보며 발생할 수 있는 시행착오(좌표계 오류, 인코딩 문제, 결측치 처리 등)를 사전에 통제하고, 데이터 정형화 규격을 맞추기 위해 작성되었습니다.

## 📋 2. 데이터 처리 5대 체크리스트 및 표준 처리 규칙

데이터를 정제할 때 아래 5가지 규칙을 반드시 적용하여 코드를 작성해 주세요.

1. **파일 인코딩 표준화**: 서울시 및 공공데이터포털 자료는 주로 `EUC-KR(CP949)`로 인코딩되어 있습니다. 로딩 시 `encoding='utf-8'` 또는 `encoding='cp949'`를 확인하여 깨짐 현상을 해결하십시오.
2. **시범 자치구(용산구) 필터링**: 전국의 방대한 데이터를 가볍게 만들기 위해 주소 컬럼에 **"용산구"**가 포함된 행만 필터링합니다. (예: `df[df['도로명주소'].str.contains('용산구', na=False)]`)
3. **좌표계 투영 변환 (`EPSG:4326` 표준화)**: 
   * 지적도나 지도 데이터(SHP)는 보통 `EPSG:5186` 또는 `EPSG:5174` 좌표계입니다.
   * 최종 적재 및 시각화는 위경도(`EPSG:4326`)를 기준으로 통일하므로 GeoPandas의 `.to_crs(epsg=4326)`를 사용하여 좌표계를 통일합니다.
4. **결측치 정제 및 이상치 드롭**: 위도/경도가 비어 있거나, 범위를 벗어난 데이터(위도 37.5, 경도 126.9 주변을 벗어나는 좌표)는 과감히 드롭(Drop) 처리합니다.
5. **격자 매핑 준비**: 수집된 위경도 핀 데이터가 격자(Grid) 내에 포함되는지 확인하기 위한 고유 ID(예: `Grid_ID`)를 매핑할 준비를 합니다.

## 🐍 3. 용산구 실증 데이터 처리 실습 코드 템플릿

팀원들이 복사해서 즉시 테스트해볼 수 있는 파이썬 전처리 예시 코드입니다. 예시 데이터는 가장 중요한 **"용산구 담배꽁초 상습 무단투기 지역 현황"** 데이터셋을 기준으로 작성되었습니다.

```python
import pandas as pd
import geopandas as gpd
from shapely.geometry import Point

def preprocess_yongsan_dumping_data(file_path):
    print("=== [1] 데이터 로드 및 인코딩 확인 ===")
    try:
        # 공공데이터포털 파일은 대개 cp949 인코딩 사용
        df = pd.read_csv(file_path, encoding='cp949')
    except UnicodeDecodeError:
        df = pd.read_csv(file_path, encoding='utf-8')
    
    print(f"원천 데이터 로드 완료: {len(df)}건")
    
    print("\n=== [2] 시범 자치구(용산구) 필터링 및 클렌징 ===")
    # 주소에 '용산구'가 들어가는 데이터만 정밀 필터링
    df_filtered = df[
        df['도로명주소'].str.contains('용산구', na=False) | 
        df['지번주소'].str.contains('용산구', na=False)
    ].copy()
    
    print(f"용산구 관내 데이터 필터링 완료: {len(df_filtered)}건")
    
    # 텍스트 주소만 있고 위경도가 없는 경우, 지오코딩 API 등을 타야 하므로 우선 팩트 파악
    # 만약 기존에 위경도 컬럼이 있다면 좌표 정제
    if '위도' in df_filtered.columns and '경도' in df_filtered.columns:
        df_filtered = df_filtered.dropna(subset=['위도', '경도'])
        df_filtered = df_filtered[(df_filtered['위도'] > 37.0) & (df_filtered['경도'] > 126.0)]
        print(f"좌표 이상치 정제 완료: {len(df_filtered)}건")
    else:
        print("[!] 본 데이터셋은 위경도 컬럼이 누락되어 지오코딩(주소 -> 좌표 변환)이 필요합니다.")
        # 브이월드 API 또는 카카오 로컬 API 활용 필요 명시
        
    return df_filtered

# 실습 가동 (각자 다운로드한 파일 경로 지정)

```

## 🤝 4. [공유 양식] 오늘 오후 4시 회의 제출 템플릿

각 팀원은 본인이 가공해 본 데이터셋에 대해 **아래 양식을 복사하여 회의록(또는 Slack/Notion)에 공유**해 주시기 바랍니다.

```markdown
### 📊 [팀원 이름 / 역할] 데이터 처리 실증 결과 보고

*   **담당 데이터셋 명칭**: (예: 서울시 어린이집 정보)
*   **데이터 원천 소스 경로**: (다운로드 링크 혹은 API 엔드포인트)
*   **데이터 형식**: (CSV / GeoJSON / SHP / API 호출)
*   **처리 결과 요약**:
    *   원천 데이터 건수: 000 건
    *   용산구 필터링 후 건수: 000 건
    *   위경도 매핑 성공률 (%): 00% (지오코딩 성공 건수 등)
*   **발견된 품질 이슈 및 특이사항**:
    *   *예시 1*: 어린이집 데이터 중 일부 위경도 값이 누락되어 도로명주소 기준으로 재변환이 필요함.
    *   *예시 2*: SHP 파일의 좌표계가 EPSG:5186으로 되어 있어 EPSG:4326으로 수동 변환 코드 추가함.
*   **DB 적재 시 고려사항**:
    *   해당 테이블은 1단계 규제 배제(Hard Constraint) 테이블이므로 지적도 폴리곤과의 GIS 차집합 연산 시 공간 인덱싱(Spatial Index) 설정 필수.
```

> [!TIP]
> **지오코딩(Geocoding) 팁**: 주소만 있고 좌표가 없는 경우, 브이월드 오픈 API를 호출하는 헬퍼 함수를 구현하거나 무료 지오코딩 툴을 사용하여 위경도를 선제적으로 뽑아낸 후 공간 알고리즘에 결합해야 합니다.
